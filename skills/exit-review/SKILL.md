# 卖出决策审查：比买入更难的那一半

对 $ARGUMENTS 执行卖出决策审查。

**支持输入格式**：
- `公司名` — 自动读取 `reports/{公司名}-thesis.md` 与最近研究报告作为输入
- `公司名 减仓/清仓/要不要卖` — 带明确诉求
- `公司名 红线触发` — thesis-tracker 红线触发后的退出方案制定

> "卖出只有三个理由：1. 发现买错了；2. 公司基本面变了；3. 找到了更好的。" —— 段永平
>
> "如果你不打算持有一家公司十年，就不要持有它十秒钟。但当事实改变时，我改变想法。" —— 综合巴菲特/凯恩斯

## 设计理念

买入有完整流程（研究→Checklist→建仓），但卖出往往是情绪驱动：跌多了恐慌卖、涨多了手痒卖、横盘久了不耐烦卖。本流程强制卖出决策走和买入同样严格的框架：**只允许三种合法卖出理由，其余一律视为情绪**。

**不适用**：还没持有、只是想研究 → 用 `investment-research`；只想检查论文是否成立 → 用 `thesis-tracker`（本流程是其下游：论文受损/破裂后来这里决定怎么退）。

## 执行流程

### 第一步：加载持仓上下文

1. 读取 `reports/{公司名}-thesis.md`（核心论文、红线清单、估值锚点、追踪记录）
2. 读取 `reports/{公司名}/` 下最近的研究/财报报告（如有）
3. 向用户确认：持仓成本、当前仓位占比、持有时长、本次想卖的原因（原话记录）

**若无投资论文**：不拒绝执行，但标注"无基线论文，卖出分析基于当前重建的逻辑"，并建议事后补建 `thesis-tracker` 论文。

### 第二步：卖出理由归类（核心步骤）

把用户的卖出冲动归入四类之一，**只有前三类是合法理由**：

| 类别 | 定义 | 验证方式 |
|------|------|---------|
| ① 买错了 | 当初的买入论文存在事实性错误（看错生意/看错人/算错账） | 逐条对照原论文5句话，指出哪句当初就不成立 |
| ② 基本面变了 | 论文当初成立，但护城河/管理层/行业发生可验证的恶化 | 引用 `thesis-tracker`/`thesis-drift` 的假设状态与红线记录 |
| ③ 有更好的去处 | 存在预期回报显著更高（税后、含转换摩擦）的替代标的或现金需求 | 第四步机会成本对比，替代标的必须已通过 `investment-checklist` |
| ④ 情绪/价格波动 | 股价涨跌、横盘、他人观点、宏观恐慌，但①②③均不成立 | **不构成卖出理由**，明确告知用户并给出应对（重读论文/降低看盘频率） |

**数据验证**：涉及的财务与估值数字按 `skills/financial-data/SKILL.md` 规范用 `tools/financial_rigor.py` 验算，禁止心算。

### 第三步：估值状态核查

卖出决策必须区分"生意变了"和"只是贵了/便宜了"：

**实时取数（强制）**：卖出决策不得使用缓存行情，先加 `--no-cache` 直连取当前价格/市值（`skills/financial-data/SKILL.md` 缓存规则）：

```bash
python3 tools/ashare_data.py quote {代码} --no-cache   # A股 600519 / 港股 hk00700 / 美股 usAAPL
```

```bash
python3 tools/financial_rigor.py verify-valuation \
  --price {现价} --eps {EPS} --bvps {每股净资产} --fcf-per-share {每股FCF}
python3 tools/financial_rigor.py three-scenario \
  --price {现价} --eps {EPS} --shares {股本亿} \
  --growth {乐观} {中性} {悲观} --pe {乐观PE} {中性PE} {悲观PE}
```

| 状态 | 判定 | 对卖出决策的含义 |
|------|------|----------------|
| 严重高估（现价 > 乐观情景目标价） | 泡沫区 | 即使论文完好，也可考虑减仓兑现（巴菲特式"价格透支未来"） |
| 合理区间 | 持有区 | 仅当①②③成立才卖 |
| 低估 | 加仓区 | 此时卖出需要极强的①②类证据（红线级） |

### 第四步：机会成本对比（仅③类理由需要）

| 维度 | 当前持仓 | 替代标的 |
|------|---------|---------|
| 预期年化回报（FCF Yield + 增速） | | |
| 确定性（生意/管理层/估值三项各1-5） | | |
| 转换摩擦（税、手续费、时间成本） | — | 估算为收益率的减项 |
| 预期回报 × 确定性（扣除摩擦后） | | |

**门槛**：替代标的扣除摩擦后的（预期回报×确定性）须**明显高于**当前持仓（建议 ≥ 1.3 倍），否则判定"换仓无意义，属于折腾"。

### 第五步：退出方案设计

若判定应卖，给出纪律化方案，不搞"一把梭"：

| 要素 | 方案 |
|------|------|
| 卖出比例 | 清仓 / 减至X% / 减半（对应理由强度：红线→清仓；单假设受损→部分） |
| 节奏 | 一次性（红线/造假类）或分2-3批（估值/换仓类） |
| 触发与撤销条件 | 什么新证据出现则停止卖出/回补 |
| 税务与流动性提示 | 持有期税率差异、大额卖出的冲击成本（提示用户自查，不给税务建议） |
| 复盘记录 | 卖出理由、当时价格与估值，写入论文文件供未来复盘 |

### 第六步：输出决策备忘录

```
一、结论（卖出/减仓/不卖，一句话+理由类别①②③④）
二、卖出理由审查表（四类归类+证据）
三、估值状态核查（工具输出）
四、机会成本对比（如适用）
五、退出方案（比例/节奏/撤销条件）
六、反面论据：不卖的最强理由是什么？
七、执行后的复盘计划
```

**结论必须明确**：卖出（比例+节奏）/ 继续持有 / 属情绪波动不构成卖出理由。不允许"可以考虑适当减仓"这类含糊表述。

### 第七步：更新文件

- 决策备忘录写入 `reports/{公司名}/{公司名}-exit-{YYYYMMDD}.md`
- 在 `reports/{公司名}-thesis.md` 的追踪记录表中追加本次决策
- 若清仓：论文文件标注"已退出（日期/价格/理由）"，保留供复盘

## 关键原则

- **只有三个合法卖出理由** — 价格波动本身永远不是理由（严重高估除外，那属于"价格透支了论文"）
- **卖出决策与买入同权** — 同样需要数据验证、反面论据和明确结论
- **红线优先** — 诚信类红线触发时，不做机会成本计算，直接退出
- **不因怕错过而不卖，不因已亏损而死扛** — 沉没成本不参与决策
- **每次卖出都留复盘记录** — 卖错和买错一样值得学习
