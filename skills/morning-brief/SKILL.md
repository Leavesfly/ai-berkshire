# 晨报：信号融合与优先级排序

对 $ARGUMENTS（可为空=全量扫描，或指定"只看观察清单"/"只看内幕交易"等侧重）执行每日信号扫描与融合。

> **定位**：不是四份原始清单的堆叠，是**信号融合**——本技能的核心价值在最后一步：用投资判断力把 N 个原始信号压缩成"今天只有 1-3 件事值得你花时间"。宁可输出"今天无事发生"，也不制造焦虑。

## 核心纪律

1. **排序即判断**——信号按"对持仓/观察标的的论文影响程度"排序，不按时间或来源排序
2. **降噪优先**——90% 的日常波动是噪音；晨报的价值是过滤，不是罗列
3. **无事如实说**——没有值得关注的信号时，输出"✅ 今日无重要信号"，禁止硬凑
4. **信号可追溯**——每条入选信号附工具输出来源，不引入未经验证的传闻

## 执行流程

### 第一步：四路信号采集（并行执行）

```bash
# 1. 观察清单价格信号（买入区/卖出区/接近触发）
python3 tools/watchlist.py scan

# 2. 决策日志持仓标的现状（含超额收益背离信号）
python3 tools/decision_log.py review --benchmark

# 3. 内幕交易异动（对观察清单+决策日志中的每个标的）
python3 tools/insider_trading.py recent {代码} --days 14

# 4. 大师持仓变动（13F 季度更新窗口内才跑，2/5/8/11 月中旬后）
python3 tools/masters_portfolio.py diff berkshire --quarters 2
```

**采集范围**：观察清单标的 + 决策日志中结论为"买入/持有"的标的（去重合并）。两者皆空时告知用户："暂无关注标的，先做一次研究或添加观察清单。"

**降级约定**：某路信号工具失败（网络/数据源）→ 跳过该路并在晨报中标注"今日 XX 信号不可用"，不中断整体流程。

### 第二步：新闻快扫（可选，标的 ≤5 个时执行）

对第一步中触发了价格/内幕信号的标的（不是全部标的），WebSearch 近 48 小时关键新闻，每个标的最多 2 条。无 WebSearch 能力时跳过并标注。

### 第三步：信号融合与分级（核心）

对所有原始信号按论文影响程度分级：

| 级别 | 判据 | 处理 |
|------|------|------|
| 🔴 **行动级** | 触发论文红线（如：多高管同期减持 + 股价破位）、观察清单进入买入区、持仓标的出现基本面事件 | 晨报置顶，给出明确下一步动作 |
| 🟡 **关注级** | 单一信号异动但不成立交叉验证（如：单笔减持、接近但未进入买卖区、大师小幅调仓） | 晨报列出，标注"继续观察，暂不行动" |
| ⚪ **噪音级** | 日常波动、可解释的常规交易（行权/缴税类减持）、与持仓无关的市场新闻 | 不进晨报正文，只计数（"另有 N 条噪音级信号已过滤"） |

**交叉验证加权**：同一标的的多路信号互相印证时升级（如：价格进入卖出区 + 内幕人减持 → 🔴）；单一来源孤立信号降级处理。

### 第四步：输出晨报

对话直接输出（默认不落盘；用户说"存档"时写入 `reports/morning-brief-{YYYYMMDD}.md`）：

```markdown
# 晨报 {YYYY-MM-DD}

📌 **今日一句话**：{最重要的一件事，或"今日无重要信号"}

## 🔴 需要行动（{N} 件）
| 标的 | 信号 | 交叉验证 | 建议动作 |
|------|------|---------|---------|
| {公司} | {信号描述} | {是/单一来源} | 说"{下一步流程触发语}" |

## 🟡 继续观察（{N} 件）
| 标的 | 信号 | 为什么暂不行动 |
|------|------|---------------|

## 数据面速览
- 观察清单：{N} 只，{X} 只接近买入区
- 持仓跟踪：{N} 只，方向一致率 {X}%
- 已过滤噪音级信号：{N} 条

## 信号缺口
{今日哪路信号不可用/未扫描，如"13F 非更新窗口，未扫描大师持仓"}
```

### 第五步：完成后引导（按生命周期只推荐 1 个）

| 触发信号 | 推荐下一步 |
|---------|-----------|
| 观察清单进入买入区 | "{公司} 进入买入区，说'过一遍买入前清单'做最后核对" |
| 持仓标的内幕人异常减持 | "说'{公司} 的管理层靠谱吗'启动管理层重审" |
| 持仓标的价格大幅背离 | "说'{公司} 为什么大跌'启动异动归因" |
| 大师大幅调仓涉及关注标的 | "说'巴菲特为什么买/卖 {公司}'查看 13F 详情" |
| 无信号 | 不引导，保持安静 |

## 注意事项

1. **快胜过全**——晨报目标 3 分钟内出结果；深度分析交给下游技能
2. **免抽检**：本技能为快反/对话级产出，按 `CLAUDE.md` 抽检政策豁免；但所有数字必须来自工具输出，禁止心算
3. **不做预测**——晨报只报告信号与事实，不预测今日涨跌
4. **定时化**：用户想每天自动收晨报时，指引其用 `watchlist.py schedule` 生成定时配置 + webhook 推送（复用 watchlist-monitor 的推送通道）
5. **与 watchlist-monitor 的区别**：watchlist-monitor 只扫观察清单价格区间；morning-brief 融合四路信号（价格 + 内幕 + 13F + 持仓复盘）并做优先级判断
