# 系统自审：让系统越用越聪明

对 $ARGUMENTS（可为空=全面自审，或指定维度如"校准"/"错误模式"）执行系统级元认知审查。

> **定位**：这是"技能的技能"——不研究公司，研究**系统自身的判断质量**。
> 桥水式循环：制度化复盘 → 发现错误模式 → 修订原则 → 未来判断更准。

## 核心纪律

1. **不粉饰**——自审的价值在于找系统性偏差，禁止只呈现正确案例
2. **样本量诚实**——决策 < 10 条时明确标注"样本不足，结论无统计意义"
3. **提议而非强改**——输出修改建议供用户确认，不自动修改任何文件
4. **区分运气与能力**——结果对≠过程对，结果错≠过程错（短期价格噪声）

## 执行流程

### 第一步：拉取校准数据

```bash
# 概率校准统计（Brier 分数 + 分桶校准）
python3 tools/decision_log.py calibrate

# 决策复盘（含基准对比）
python3 tools/decision_log.py review --benchmark

# 原始记录（含理由与概率）
python3 tools/decision_log.py list --limit 50
```

无记录时如实告知："决策日志为空，完成任意研究流程后会自动累积"，并结束。

### 第二步：三维诊断

| 维度 | 检查内容 | 数据来源 |
|------|---------|---------|
| **校准偏差** | 系统是否系统性过度自信/过度谨慎？哪个概率桶偏差最大？ | calibrate 输出 |
| **错误模式** | 错误决策有无共性？（高估成长/低估风险/信息不足/行业偏见） | review + list |
| **流程盲区** | 哪类标的/哪种子流程的准确率最低？是否有系统性弱点？ | 按 skill/verdict 分组统计 |

### 第三步：输出自审报告

短报告（≤10 条记录）直接对话输出；长报告写入 `reports/self-review-{YYYYMMDD}.md`：

```markdown
# 系统自审报告

**自审日期**：{日期} | **覆盖决策**：{N}条 | **Brier 分数**：{X}

## 一、校准诊断
| 概率桶 | 样本 | 平均预测 | 实际发生 | 偏差 | 诊断 |
|--------|------|---------|---------|------|------|

**校准结论**：{过度自信/校准良好/过度谨慎}

## 二、错误模式归纳
（如样本足够，归纳 1-3 条系统性教训）

| 模式 | 出现次数 | 典型案例 | 根因分析 |
|------|---------|---------|---------|

## 三、流程盲区
| 子流程 | 决策数 | 方向一致率 | 诊断 |
|--------|--------|-----------|------|

## 四、修改建议（供用户确认）

### 建议 1：{标题}
- **问题**：{描述系统性偏差}
- **证据**：{数据支撑}
- **建议修改**：{具体修改哪个文件的哪个规则}
- **预期效果**：{修改后如何改善}

### 建议 2：...

## 五、无需修改的方面
（明确标注哪些维度表现良好，避免过度修正）
```

### 第四步：修改建议落地（用户确认后）

用户确认某条建议后，执行对应修改：

| 建议类型 | 修改目标 | 示例 |
|---------|---------|------|
| 概率标注习惯 | `references/report-conventions.md` 概率化结论规范 | "系统偏乐观 → 追加约束：成长股概率上限 75%" |
| 检查清单 | `skills/investment-checklist/SKILL.md` | "连续高估护城河 → 追加'护城河变窄信号'检查项" |
| 否决条件 | `references/masters-profiles.md` | "管理层减持误判 → 细化减持解读纪律" |
| 流程调整 | 对应子流程 SKILL.md | "C级信息标的准确率低 → 强化第一性原理模式约束" |

> **安全约束**：每次最多提议 3 条修改；修改 masters-profiles.md 的否决条件需用户二次确认（这是最高优先级约束文件）。

### 第五步：完成后引导

- 校准偏差明显 → "建议后续研究时注意概率标注，下次自审时验证改善"
- 发现错误模式 → "建议对同类标的增加 {XX} 维度的审查权重"
- 样本不足 → "继续积累决策记录，≥10 条后自审结论才有统计意义"

## 注意事项

1. **元认知而非预测**——本流程不预测市场，只诊断系统自身
2. **时间权重**：3 个月内的决策标注"验证期过短"；价值投资以年为单位验证
3. **不自动执行修改**——所有修改建议必须经用户确认，禁止静默改动规则文件
4. **免抽检**：本流程为对话/回顾级产出，按 `CLAUDE.md` 快反档豁免抽检
5. **与 track-record 的区别**：track-record 复盘单次决策对错；self-review 诊断系统性偏差并提议规则修改
