# 决策复盘：检验历史判断，让系统越用越聪明

对 $ARGUMENTS（可为空=全部决策，或指定公司名）执行决策复盘。

> "如果你不知道自己错在哪，你就会一直错下去。" —— 芒格式反馈闭环：
> 研究流程负责"每次都聪明"，本流程负责"越用越聪明"。

## 数据来源

决策日志 `data/decisions.jsonl` 由各研报级流程在交付结论后自动追加（约定见 `CLAUDE.md`「决策日志约定」）。每条记录含：日期、公司、子流程、结论（买入/观望/回避/卖出等）、当时价格、核心理由、关联报告路径。

## 执行流程

### 第一步：拉取复盘基础数据

```bash
# 全部决策 vs 现价（自动取现价，命中缓存秒回）
python3 tools/decision_log.py review

# 含同期基准对比（按市场自动选 沪深300/恒指/标普500；胜率高但超额为负=只是赶上了牛市）
python3 tools/decision_log.py review --benchmark

# 指定公司
python3 tools/decision_log.py review --company 腾讯

# 查看原始记录（含理由与关联报告）
python3 tools/decision_log.py list --limit 50
```

无记录时如实告知："决策日志为空，完成任意一次研究流程后会自动累积"，并结束流程。

### 第二步：分层解读（不只看价格）

价格涨跌只是**初步信号**，逐条深入到判断质量：

| 层次 | 问题 | 数据来源 |
|------|------|---------|
| 方向 | 结论与后续走势是否一致？ | review 输出 |
| 逻辑 | 当时的核心理由如今被证实还是证伪？ | 记录中的 reason + WebSearch 近况 |
| 过程 | 即使结果对了，推理过程是否靠运气？（结果偏差检验） | 关联报告原文 |
| 教训 | 错误决策的共同模式是什么？（高估成长/低估风险/信息不足…） | 跨记录归纳 |

对「⚠️ 暂时背离」的标的：区分**逻辑证伪**（应提示走 `thesis-drift` / `exit-review`）与**市场情绪波动**（论文未变，继续持有判断）。

### 第三步：输出复盘报告

短复盘（≤5 条记录）直接对话输出；长复盘写入 `reports/track-record-{YYYYMMDD}.md`：

```markdown
# 决策复盘报告

**复盘日期**：{日期} | **覆盖决策**：{N}条 | **时间跨度**：{最早}~{最新}

## 一、总览
| 结论类型 | 条数 | 方向一致 | 一致率 |
|---------|------|---------|--------|

## 二、逐条复盘（价格信号 + 逻辑验证）
| 日期 | 公司 | 结论 | 当时价→现价 | 核心理由 | 逻辑现状 | 判定 |
|------|------|------|------------|---------|---------|------|
（判定：判断正确 / 暂时背离-逻辑未变 / 逻辑证伪 / 结果对但过程存疑）

## 三、错误模式归纳
（如样本足够，归纳 1-3 条共性教训，例如"对高估值成长股的增速预期系统性偏乐观"）

## 四、行动项
（需要复查论文的标的 → thesis-drift；需要退出审查的 → exit-review）
```

### 第四步：完成后引导

- 存在「逻辑证伪」标的 → "建议对 {公司} 说'论文还成立吗'启动复查"
- 一致率明显偏低的结论类型 → 提示在后续同类研究中增加对应反偏见检查

## 注意事项

1. **样本量诚实**：决策 < 10 条时明确标注"样本不足，一致率无统计意义"。
2. **时间权重**：3 个月内的决策标注"验证期过短"；价值投资以年为单位验证。
3. **不粉饰**：复盘的价值在于找错误，禁止只呈现正确案例；错误案例必须逐条给出归因。
4. **免抽检**：本流程为对话/回顾级产出，按 `CLAUDE.md` 快反档豁免抽检，但价格数据须来自 decision_log review 的工具输出（禁止心算涨跌幅）。
