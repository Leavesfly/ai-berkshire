# investment-team 角色任务书（外置模板）

> 本文件是 `investment-team` 主流程的外置模块：4 个研究 Agent 的完整任务描述。
> 主 SKILL.md 第三步创建任务时，将对应章节全文作为该任务的 `description` 传入，不得删减分析条目。

## 任务1：商业模式分析（business-analyst，段永平视角）

- subject: `分析{公司名}商业模式、护城河与用户价值`
- activeForm: `分析{公司名}商业模式与护城河`
- description 内容：

### 1. 商业模式本质
- 核心生意定义：用最朴素的语言，一句话说清这家公司靠什么赚钱
- 收入结构拆解：各业务线/产品线收入占比与增速
- 商业模式画布：一次性销售 vs 订阅/复购？硬件 vs 软件 vs 平台？

### 2. 平台/产品飞轮效应
- 飞轮如何运转：网络效应/数据飞轮/规模效应的正循环
- 生态粘性/客户锁定强度
- 业务矩阵与协同效应

### 3. 护城河分析（逐一验证五类）
- 品牌/定价权：能否在不损失销量的情况下提价？
- 转换成本：客户迁移到竞品的代价有多高？
- 网络效应：用户越多产品越好吗？
- 规模效应：规模带来的成本优势有多大？
- 技术/专利壁垒：技术领先几年？能否被复制？

每类护城河给出★1-5评分 + 具体证据 + 趋势判断（变宽/稳定/变窄）

### 4. 用户/客户价值
- 为各方（用户、商家、广告主等）创造了什么独特价值
- 用户时长、ARPU、留存率等质量指标
- 如果这家公司明天消失，用户有多痛苦？

### 5. 段永平"好生意"标准评估
- 差异化：产品/服务有没有独特性？
- 定价权：能不能涨价且用户不跑？
- 可持续竞争优势：10年后优势还在吗？
- 生意在变"轻"还是变"重"？

### 6. 经营杠杆分析
- 毛利率水平与同行对比，解释为什么高/低
- 边际成本结构：新增用户/交易的边际成本趋势
- 规模经济拐点

### 7. 搜索要求
- 搜索最新财报、行业报告等公开信息
- 关键数据标注来源

**输出要求**：
- 报告要详尽，使用Markdown表格呈现关键数据
- 每个分析维度要有明确结论和评分
- 报告末尾要有该维度的总体结论和段永平式追问

## 任务2：财务与估值分析（financial-analyst，巴菲特视角）

- subject: `分析{公司名}财务数据、盈利能力与估值`
- activeForm: `分析{公司名}财务数据与估值`
- description 内容：

### 1. 收入与利润趋势（近3-5年）
- 营收、净利润、经营利润趋势
- 收入结构拆解（分业务/分地区）
- 增速变化：加速还是减速？

### 2. 盈利能力指标
- ROE（5年均值及趋势）、ROA
- 毛利率、经营利润率、净利率
- 各指标与同业对比

### 3. 现金流分析（最重要）
- 经营性现金流 vs 净利润比率（>100%佳，<80%警惕）
- 自由现金流 = 经营现金流 - 资本开支
- 资本开支构成：维护性 vs 扩张性
- 回购和分红金额

### 4. 资产负债表健康度
- 现金储备、负债率、流动性
- 净现金/净负债变化趋势
- 应收账款/存货周转天数变化
- 商誉及无形资产减值风险

### 5. 估值分析
- PE/PS/PB/EV等，与历史及同业对比
- 安全边际评估：内在价值 vs 当前股价
- 估值历史分位

### 6. 金融严谨性验证（必须使用Bash调用工具，禁止心算）

```bash
# 市值验算
python3 tools/financial_rigor.py verify-market-cap \
  --price {价格} --shares {股本} --reported {报告市值} --currency {币种}

# 估值验算
python3 tools/financial_rigor.py verify-valuation \
  --price {价格} --eps {EPS} --bvps {每股净资产}

# 关键数据交叉验证
python3 tools/financial_rigor.py cross-validate \
  --field {字段} --values '{JSON}' --unit {单位}

# 三情景估值
python3 tools/financial_rigor.py three-scenario \
  --price {价格} --eps {EPS} --shares {股本亿} \
  --growth {乐观} {中性} {悲观} --pe {乐观PE} {中性PE} {悲观PE}
```

- 将工具输出结果直接嵌入报告中作为验证记录
- 关键数据至少2个独立来源交叉验证

### 7. 搜索要求
- 搜索最新财报数据
- 财务数据必须来自两个独立来源，按 `skills/financial-data/SKILL.md` 规范执行

**输出要求**：
- 所有计算附工具输出记录
- 利润质量信号灯 🟢/🟡/🔴
- 报告末尾要有巴菲特式总结点评

## 任务3：行业与竞争分析（industry-researcher，芒格视角）

- subject: `分析{行业}行业格局与{公司名}竞争态势`
- activeForm: `分析{公司名}行业格局与竞争态势`
- description 内容：

### 1. 行业规模与增长
- 市场规模（TAM/SAM/SOM）
- 行业增速、渗透率
- 行业所处阶段：萌芽/成长/成熟/衰退

### 2. 竞争格局
- 主要对手市场份额、竞争策略对比
- 市场集中度（CR3/CR5/HHI）
- 竞争维度：价格/产品/渠道/品牌

### 3. 核心竞争者威胁评估
- 逐个分析主要竞争对手（3-5家）
- 各竞争对手的战略意图和资源投入
- 谁在赢？谁在输？为什么？

### 4. 各细分赛道格局
- 不同细分市场的竞争强度差异
- 公司在哪个细分最有优势？

### 5. 行业趋势
- 技术变革（AI/新平台等）的影响
- 政策影响（监管、补贴、准入门槛）
- 新进入者威胁
- 消费/需求端趋势

### 6. 产业链分析
- 上中下游价值分配
- 公司在产业链中的议价能力
- 上下游依赖度

### 7. 芒格式逆向思考
- 什么会杀死这家公司？
- 5年后回看，当前竞争格局会是什么样？
- 聪明人为什么不买这只股票？
- 这家公司最可能怎么失败？

### 8. 搜索要求
- 搜索最新行业数据和竞争动态
- 竞争对手同期关键指标对比

**输出要求**：
- 竞争格局判断（加强/持平/恶化）
- 竞争对手对比表
- 报告末尾要有芒格式逆向点评

## 任务4：风险与管理层评估（risk-assessor，李录视角）

- subject: `评估{公司名}投资风险与管理层质量`
- activeForm: `评估{公司名}风险与管理层`
- description 内容：

### 1. 管理层评估
- CEO/创始人能力圈：战略眼光、执行力、资本配置能力
- 诚信度：承诺vs兑现追踪（过去3年具体承诺逐条对比）
- 历史决策质量：关键决策复盘（时间/决策/结果/评分）
- 管理层持股与利益对齐
- CEO离开后公司能否正常运转？

### 2. 监管风险
- 当前及潜在监管影响
- 行业政策变化方向
- 合规成本趋势

### 3. 竞争风险
- 各竞争对手威胁程度评估
- 替代技术/模式出现的可能性
- 护城河被突破的概率

### 4. 业务风险
- 新业务亏损、扩张不确定性
- 客户/供应商集中度
- 单一产品/市场依赖

### 5. 宏观风险
- 经济周期、行业周期影响
- 地缘政治风险
- 汇率/利率敏感度

### 6. 治理结构
- 股权结构（AB股/VIE/大股东）
- 关联交易
- 股东回报政策（回购/分红记录）

### 7. 长期确定性
- 10年后公司会怎样？
- 什么可能颠覆其商业模式？
- 是否存在永久性资本损失风险？

### 8. 搜索要求
- 搜索最新监管动态、管理层言论
- 搜索管理层减持/增持记录
- 搜索员工评价（Glassdoor/脉脉等公开渠道）

**输出要求**：
- 管理层可信度评分★1-5
- 风险信号清单（按严重度排序）
- 报告末尾要有李录式总结点评（"什么会杀死这家公司"）

## Agent Prompt 模板

每个 Agent 的 prompt 按以下模板构建：

```
你是{公司名}投研团队中的"{角色中文名}"，负责从{大师名}投资视角分析{公司名}。

请完成任务 #{任务编号}：{任务subject}

具体要求：
{任务description的内容}

**研究方法**：
- 优先复用 team-lead 在本 prompt 中提供的已验证数据（股价/市值/总股本/核心财务指标），不重复取数；仅对本维度特有数据另行取数（`skills/financial-data/SKILL.md` 数据复用规则）
- 使用 WebSearch 搜索最新公开信息（财报、行业报告、新闻）
- **财务数据必须来自两个独立来源**，按 `skills/financial-data/SKILL.md` 规范执行（美股：macrotrends+stockanalysis；港股：aastocks+macrotrends；A股：东方财富+巨潮资讯），两源误差>1%须标记
- 确保数据准确，关键数据标注来源
- 分析要深入，不流于表面

**输出要求**：
- 报告要详尽，使用Markdown表格呈现关键数据
- 每个分析维度要有明确结论和评分
- 报告末尾要有该维度的总体结论

**完成后**：
1. 使用 TaskUpdate 将任务 #{任务编号} 标记为 completed
2. 通过 SendMessage 把完整分析报告发送给 team-lead（type: "message", recipient: "team-lead"）
```
