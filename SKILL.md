# AI Berkshire 投研技能：意图识别与子流程调度

这是 AI Berkshire 投研能力的**统一入口**。核心职责：**识别用户的投研意图 → 选定最合适的子流程 → 读取并严格执行 `skills/<名称>/SKILL.md` → 传递参数并启动**。

四位价值投资大师（巴菲特、芒格、段永平、李录）方法论系统化，覆盖从个股研究到内容创作的全流程。所有分析遵循本技能根目录 [`CLAUDE.md`](CLAUDE.md) 的客观性原则。

> **执行约定**：下表"子流程"列的每个名称，都对应本技能目录下的一个文件 `skills/<名称>/SKILL.md`。选定后，**打开该文件并严格按其流程执行**。工具脚本位于 `tools/`，均以本技能根目录为基准路径调用（如 `python3 tools/financial_rigor.py ...`）。

> **入参约定**：本入口的原始输入为 `$ARGUMENTS`（用户调用时传入的研究请求，如公司名、行业、意图描述）。
> - `$ARGUMENTS` **非空**：将其作为用户意图，按下方路由表匹配子流程，选定后把 `$ARGUMENTS` 原样传递给对应的 `skills/<名称>/SKILL.md` 启动。
> - `$ARGUMENTS` **为空**：不臆测意图，只呈现下方「分层新手引导」的 **L0 极简引导**（不要输出 L1/L2），请用户说明研究对象与目标后再路由。

## 三种调用方式与路由优先级

| 方式 | 触发形式 | 处理逻辑 |
|------|---------|---------|
| **自然语言** | "帮我研究一下美团"、"这家公司财报怎么样" | 按下方路由表匹配意图，按「分级确认机制」决定是否确认后启动 |
| **显式指定** | "用 investment-team 研究美团" | 直接读取 `skills/investment-team/SKILL.md` 执行，无需确认 |
| **菜单** | "AI Berkshire 能做什么"、"有哪些能力" | 展示「分层新手引导」的 L1 能力菜单，让用户选择 |

**路由优先级**（冲突时从上到下）：
0. 一票路由（强信号直接短路）：命中未上市公司名（蚂蚁/SpaceX/小红书等）→ `private-company-research`；命中"写文章/公众号/长文系列"→ 内容创作类；命中"要不要卖/减仓/清仓"→ `exit-review`；
1. 用户显式指定的子流程名 → 直接执行；
2. **报告后操作**（见下方「报告后操作路由」）→ 对已有产出的二次操作；
3. 会话上下文复用 → 当前对话已明确研究对象/持仓状态时，后续追问直接继承，不重复询问；
4. 生命周期上下文 → 用户已声明持仓/已有论文文件时，优先决策监控类；
5. 关键词匹配 → 按路由表（口语化触发词映射见 [`references/routing-guide.md`](references/routing-guide.md)）；
6. 模糊兜底 → 用户只给对象没有意图（如只说"腾讯"）→ 一句话反问："关于{对象}，你想：① 研究值不值得买 ② 看最新财报 ③ 查股价异动原因 ④ 其他（直接说）"；
7. 仍有 ≥2 个候选 → 用下方「智能澄清机制」。

> **消歧参考**：遇到易混淆意图时，查阅 [`references/routing-guide.md`](references/routing-guide.md) 的消歧规则与案例对照表。

**分级确认机制**（自然语言触发时按档位执行；显式指定一律免确认）：

| 档位 | 子流程 | 策略 |
|------|--------|------|
| **轻量（免确认）** | `quality-screen` `dyp-ask` `earnings-review` `investment-checklist` `thesis-tracker` `thesis-drift` `track-record` `watchlist-monitor` `masters-portfolio` `morning-brief` | 直接执行，开头一句话告知："已按 XX 流程处理，对象：XX" |
| **中量（一句话确认）** | `investment-research` `exit-review` `industry-research` `portfolio-review` `news-pulse` `management-deep-dive` `red-team` `self-review` | 20 字内确认（流程 + 对象），用户回“好/开始/嗯”即启动 |
| **重量（完整确认）** | `investment-team` `earnings-team` `wechat-article` `deep-company-series` `private-company-research` `industry-funnel` `bottleneck-hunter` | 使用完整确认话术（多 Agent / 长耗时，值得用户知情后再启动） |

**完整确认话术模板**（仅重量档使用）：

> 将执行 **`<子流程名>`**（<一句话说明干什么>），研究对象：<对象>；产出：`reports/...` 报告；预计耗时：<多Agent并行需要更长，会实时更新进度>。确认开始？也可以说"换成快速模式"或直接补充要求。

## 意图路由表（按投资生命周期组织）

> 分类判据：看**输入前提**（是否已持仓/已有论文）与**输出契约**（研报/决策/内容/对话）。口语化触发词的完整映射见 [`references/routing-guide.md`](references/routing-guide.md)。

### ① 发现与筛选

| 意图关键词 | 子流程 | 说明 |
|-----------|--------|------|

### ② 深度研究

## 报告后操作路由（对已有产出的二次操作）

当用户在收到报告/结论后发出以下指令时，**不重跑全流程**，按对应方式处理：

| 用户说 | 处理方式 |
|--------|---------|
| "重点看一下XX维度" / "补充海外收入分析" | 基于已有报告数据，补充指定维度分析，追加到报告或对话输出 |
| "太长了，三句话总结" / "给我摘要" | 从已有报告提取倒金字塔结论（一行结论 + 3 行数据） |
| "用英文重写" / "换个格式" | 对已有报告做格式/语言转换 |
| "上次研究XX的结论是什么" | 读取 `reports/{公司名}/` 下最新报告，输出摘要 + 路径 |
| "之前那个数据更新一下" | 仅对时效性数据（股价/市值/最新财报）重新取数，更新报告对应段落 |
| "换成团队模式重做" / "换成快速模式" | 切换子流程，复用已收集的基础数据，不从头取数 |

> 路由优先级中第 2 级即匹配此类操作——当会话中已有报告产出且用户指令指代"刚才/上次/那个"时触发。

## 智能澄清机制（意图不明时）

触发条件：路由后仍有 ≥2 个候选子流程，或缺少**必填**参数。

**参数分级（缺参时先分级，再决定问不问）**：

| 级别 | 示例 | 策略 |
|------|------|------|
| **必填** | 公司名/行业名、组合检视的持仓清单 | 缺失必须问，不带缺口执行 |
| **可推断** | 财报期、新闻时间窗口 | 用默认值执行并声明（如"已自动选取最新一期财报"） |
| **可选** | 关注侧重、输出风格 | 不问，用子流程默认值 |

> 本参数分级优先于子流程内的参数追问要求：子流程要求反问的**可推断**参数，一律改为"默认值 + 开头声明"。

**澄清规则**：
1. 最多问 2 个问题，一次性问完；
2. 每个问题给出编号选项，用户回一个数字即可；
3. 选项中标注**产出物与耗时差异**；
4. 用户说"你看着办" → 选最轻量候选并声明理由。

## 研究生命周期（标准链路）

```
筛选：industry-funnel / quality-screen / bottleneck-hunter
  → 深研：investment-research 或 investment-team（+ management-deep-dive 纵深）
    → 买前核对：investment-checklist
      → 买入后：thesis-tracker（建论文）
        → 持有期：earnings-review（每季）/ news-pulse（异动时）/ thesis-drift（定期）
          → 退出决策：exit-review（红线触发或想卖时）
组合层：portfolio-review（每季度，跨标的）
```

路由时若发现用户处于链路中段（如已持仓但无论文），主动提示补齐上游环节。

## 错误处理与用户反馈

| 场景 | 处理方式 | 向用户反馈 |
|------|---------|-----------|
| 意图不明 | 用「智能澄清机制」，最多 2 问 | 编号选项，一次问完 |
| 必填参数缺失 | 先补齐再启动；可推断参数用默认值+声明 | "需要你补充：XXX" / "已默认选取 XX" |
| 超出范围（短线择时/加密货币/期权） | 如实说明不做，给 1-2 个可行替代 | "本技能不做短线预测。不过我可以：① … ② …" |
| 用户中途改需求 | 停止当前流程，切换并复用已收集数据 | "已停止 XX，切换到 XX，数据复用" |
| 子流程中途失败 | 用已完成部分出降级版结论，标注缺失维度 | "N-1 个维度完成，XX 失败，已降级输出" |
| 工具调用失败（网络/参数） | 按 verification-playbook 降级 | 标注"未经工具验算"；**告知用户可说"重试"再试一次，或"跳过"用已有数据继续** |
| 数据不足（冷门股/新上市） | 按 C 级信息稀缺模式执行 | "信息稀缺，已切换第一性原理模式" |
| 报告写入失败 | 先创建目录重试；仍失败输出到对话 | 告知原因及建议路径 |

## 执行环境自检（首次路由后静默执行）

**首次触发子流程前**，静默运行 `python3 tools/doctor.py`：
- 退出码 0（全部就绪）→ 不告知用户，无感通过；
- 退出码 1（有降级项）→ 一句话告知："当前环境无 XX，将用 XX 方式执行，不影响结论质量"；
- 无 Python 环境 → 按下表降级，同样一句话告知。

| 检查项 | 具备时 | 缺失时降级 |
|--------|--------|-----------|
| Team 协作工具 | 按子流程原文执行 | 降级为并行子代理 |
| 并行子代理 | 并行执行 | 单 Agent 顺序执行，标注"非真正并行" |
| WebSearch/WebFetch | 正常取数 | 告知无法联网，仅基于用户提供资料 |
| Bash + Python3 | 调用 tools/ 验算 | 双源人工比对，标注"未经工具验算" |

> 同一会话只自检一次，后续子流程不重复。

## 启动、进度与完成后引导约定

1. **启动时**：按「分级确认机制」档位执行。轻量档直接开跑并一句话告知；中量/重量档确认后启动。

2. **执行中进度反馈**（所有流程适用）：
   - **轻量流程**（≤2 分钟）：启动时一行即可 → "⚡ 正在筛选 3 家公司..."
   - **多 Agent/长流程**：每完成一个阶段更新 → "📊 进度 [2/4]：{阶段名} 完成 → 核心发现：① … ② …；剩余：{阶段名}，预计还需 {X} 分钟"
   - 不要长时间无输出（>60 秒须有进度信号）。

3. **完成时（倒金字塔 + 可复制结论）**：
   - 第一行：**📌 一句话结论**（可直接复制/转发）→ 格式：`📌 {公司}——{买入/观望/回避}（{核心数据}，核心风险：{XX}）`
   - 第二行起：不超过 3 行关键数据支撑
   - 报告文件路径
   - 下一步建议（只给 1 个，见下表）
   - 简单流程可压缩到 📌 结论 + 路径两行。

4. **数据继承提示**：启动下游流程时，若复用了上游数据，一句话告知 → "已复用上次{流程名}中{公司}的基础数据（ROE/毛利率等），无需重新取数。"

5. **下一步引导表**（按生命周期只推荐 1 个）：

   | 刚完成 | 推荐下一步 |
   |--------|-----------|
   | 筛选类 | "要不要深入研究入选的 {公司名}？说'研究 {公司名}'即可" |
   | investment-research / investment-team | "考虑买入的话，说'过一遍买入前清单'做最后核对；结论为观望的话，可以'加入观察清单'等买点" |
   | investment-checklist（通过） | "买入后说'建论文'，开始持有期跟踪" |
   | earnings-review / news-pulse（发现变化） | "影响持仓判断的话，说'论文还成立吗'复查逻辑" |
   | thesis-tracker / thesis-drift（红线触发） | "考虑卖出的话，说'要不要卖'启动退出审查" |
   | exit-review | "卖出后可以说'审视我的组合'，看仓位要不要调" |
   | watchlist-monitor（买入区触发） | "{公司} 进入买入区，说'过一遍买入前清单'做最后核对" |
   | track-record（发现逻辑证伪） | "建议对 {公司} 说'论文还成立吗'复查持仓逻辑" |

6. **会话记忆摘要**：本次会话完成 ≥2 个标的的研究/操作后，在后续引导中附带一行 → "📋 本次会话已研究：{公司A}（结论）、{公司B}（结论）。继续说公司名即可。"

7. **首份报告后的视角引导**：用户首次收到报告时追加一句："报告含四大师独立视角，下次可以指定'用芒格视角重点看风险'"。后续不重复。

8. **渐进解锁提示**：每完成一类任务，最多附 1 条进阶提示（同一提示一个会话只出现一次，不与下一步引导叠加超过两条）。

9. **扩展**：新增能力只需在 `skills/` 下新建 `<名称>/SKILL.md`，并在路由表与分级确认机制中各补充一行。
