# AI Berkshire 投研技能：意图识别与子流程调度

这是 AI Berkshire 投研能力的**统一入口**。核心职责：**识别用户的投研意图 → 选定最合适的子流程 → 读取并严格执行 `skills/<名称>/SKILL.md` → 传递参数并启动**。

四位价值投资大师（巴菲特、芒格、段永平、李录）方法论系统化，覆盖从个股研究到内容创作的全流程。所有分析遵循本技能根目录 [`CLAUDE.md`](CLAUDE.md) 的客观性原则。

> **执行约定**：下表"子流程"列的每个名称，都对应本技能目录下的一个文件 `skills/<名称>/SKILL.md`。选定后，**打开该文件并严格按其流程执行**。工具脚本位于 `tools/`，均以本技能根目录为基准路径调用（如 `python3 tools/financial_rigor.py ...`）。

## 三种调用方式

| 方式 | 触发形式 | 处理逻辑 |
|------|---------|---------|
| **自然语言** | "帮我研究一下美团"、"这家公司财报怎么样" | 按下方路由表匹配意图，选定子流程后**先向用户确认**再启动 |
| **显式指定** | "用 investment-team 研究美团" | 直接读取 `skills/investment-team/SKILL.md` 执行，无需确认 |
| **菜单** | "AI Berkshire 能做什么"、"有哪些能力" | 展示下方能力菜单，让用户选择 |

## 消歧规则

当用户意图落在多个子流程之间时，按以下规则处理：

1. **研究深度**：想要"快速判断" → `investment-research`；想要"深度并行团队" → `investment-team`。不确定时反问一句。
2. **单公司 vs 行业**：明确了公司名 → 个股类；只给了赛道/方向 → 行业类。
3. **上市 vs 未上市**：目标是蚂蚁、SpaceX 等未上市公司 → 强制走 `private-company-research`。
4. **研究 vs 内容**：目标是"做投资决策" → 研究类；目标是"发文章" → 内容创作类。
5. **持仓 vs 新标的**：用户已持有并问"要不要继续拿" → 决策监控类（`thesis-tracker` / `thesis-drift` / `news-pulse`）。

## 错误处理

- **意图不明**：不要默默假设，向用户澄清 1-2 个关键问题（研究对象？想要什么深度？是否已持仓？）。
- **参数缺失**：子流程需要的公司名/行业/时间窗口等未提供时，先补齐再启动。
- **超出范围**：如果用户需求超出本技能范围，如实说明并建议最接近的替代方案，不强行套用。

## 启动约定

确认意图后，明确告知用户"将执行 `skills/<名称>/SKILL.md`"，然后打开该文件并严格按其流程运行。新增能力只需在 `skills/` 下新建 `<名称>/SKILL.md`，并在上方路由表补充一行即可扩展。
