# AI Berkshire 投研技能：意图识别与子流程调度

这是 AI Berkshire 投研能力的**统一入口**。核心职责：**识别用户的投研意图 → 选定最合适的子流程 → 读取并严格执行 `skills/<名称>/SKILL.md` → 传递参数并启动**。

四位价值投资大师（巴菲特、芒格、段永平、李录）方法论系统化，覆盖从个股研究到内容创作的全流程。所有分析遵循本技能根目录 [`CLAUDE.md`](CLAUDE.md) 的客观性原则。

> **执行约定**：下表"子流程"列的每个名称，都对应本技能目录下的一个文件 `skills/<名称>/SKILL.md`。选定后，**打开该文件并严格按其流程执行**。工具脚本位于 `tools/`，均以本技能根目录为基准路径调用（如 `python3 tools/financial_rigor.py ...`）。

> **入参约定**：本入口的原始输入为 `$ARGUMENTS`（用户调用时传入的研究请求，如公司名、行业、意图描述）。
> - `$ARGUMENTS` **非空**：将其作为用户意图，按下方路由表匹配子流程，选定后把 `$ARGUMENTS` 原样传递给对应的 `skills/<名称>/SKILL.md` 启动。
> - `$ARGUMENTS` **为空**：不臆测意图，先呈现下方「新手引导」与能力菜单，请用户说明研究对象与目标后再路由。

## 新手引导（第一次使用看这里）

> 面向初次使用或不确定该问什么的用户。**当用户输入为空、说“不知道从哪开始”、或问“怎么用”时，先把本节呈现给用户，再等其选择。**

**一句话介绍**：AI Berkshire 用巴菲特、芒格、段永平、李录四位大师的方法论，帮你判断一家公司/一个行业值不值得投、财报好不好、持仓要不要动。

**直接照抄这些话就能开始**（把公司/行业换成你关心的即可）：

| 你想做什么 | 直接说 | 会启动 |
|-----------|-------|--------|
| 研究一家公司值不值得买 | “帮我研究一下贵州茅台” | `investment-research` |
| 买之前快速过一遍清单 | “买英伟达前帮我核对一下” | `investment-checklist` |
| 快速筛掉不合格的标的 | “腾讯、美团、比亚迪哪些够格” | `quality-screen` |
| 看懂最新财报 | “读一下腾讯 2025Q4 财报” | `earnings-review` |
| 研究一个赛道 | “帮我看看创新药这个行业” | `industry-research` |
| 检视我的持仓 | “帮我审视一下我的组合” | `portfolio-review` |
| 股价大涨大跌想知道为什么 | “拼多多今天跌了8%，发生了什么” | `news-pulse` |
| 纠结要不要卖 | “腾讯涨了很多，要不要卖点” | `exit-review` |
| 写一篇公众号深度稿 | “写一篇拼多多的深度文章” | `wechat-article` |

**三步上手**：
1. 用一句话说清“研究对象 + 想要什么”（如“研究拼多多，判断现在能不能买”）。
2. 我会先确认理解是否正确，再启动对应子流程。
3. 不确定选哪个？直接说“我不确定”，我会反问 1-2 个问题帮你定位。

**新手小贴士**：
- 不用记子流程名字，用大白话描述需求即可。
- 想看全部能力，问“AI Berkshire 能做什么”。
- 越具体越好：给出**公司名/行业 + 目的**（判断买入？跟踪持仓？写文章？）能显著提升结果质量。

## 三种调用方式与路由优先级

| 方式 | 触发形式 | 处理逻辑 |
|------|---------|---------|
| **自然语言** | "帮我研究一下美团"、"这家公司财报怎么样" | 按下方路由表匹配意图，选定子流程后**先向用户确认**再启动 |
| **显式指定** | "用 investment-team 研究美团" | 直接读取 `skills/investment-team/SKILL.md` 执行，无需确认 |
| **菜单** | "AI Berkshire 能做什么"、"有哪些能力" | 展示下方能力菜单，让用户选择 |

**路由优先级**（冲突时从上到下）：
1. 用户显式指定的子流程名 → 直接执行；
2. 生命周期上下文 → 用户已声明持仓/已有论文文件时，优先决策监控类而非从头研究；
3. 关键词匹配 → 按路由表；
4. 仍有 ≥2 个候选 → 不猜，用下方「智能澄清机制」。

**确认话术模板**（自然语言触发时使用，一段话说清三件事，不要长篇大论）：

> 将执行 **`<子流程名>`**（<一句话说明干什么>），研究对象：<对象>；产出：`reports/...` 报告；预计耗时：<单Agent几分钟 / 多Agent并行需要更长，会实时更新进度>。确认开始？也可以说“换成团队模式/快速模式”或直接补充要求。

## 消歧规则

当用户意图落在多个子流程之间时，按以下规则处理：

1. **研究深度**：想要"快速判断" → `investment-research`；想要"深度并行团队" → `investment-team`。不确定时反问一句。
2. **单公司 vs 行业**：明确了公司名 → 个股类；只给了赛道/方向 → 行业类。
3. **上市 vs 未上市**：目标是蚂蚁、SpaceX 等未上市公司 → 强制走 `private-company-research`。
4. **研究 vs 内容**：目标是"做投资决策" → 研究类；目标是"发文章" → 内容创作类。
5. **持仓 vs 新标的**：用户已持有并问"要不要继续拿" → 决策监控类（`thesis-tracker` / `thesis-drift` / `news-pulse`）；问"要不要卖/卖多少" → `exit-review`。
6. **三种"筛选"的区分**：已有候选名单要过滤去劣 → `quality-screen`；给定行业要选出最优几家 → `industry-funnel`；从超级趋势找被忽视的供应链标的 → `bottleneck-hunter`。

## 智能澄清机制（意图不明时）

触发条件：路由后仍有 ≥2 个候选子流程，或缺少必要参数（研究对象/时间窗口/是否持仓）。

**规则**：
1. 最多问 2 个问题，一次性问完，不要多轮拉锯；
2. 每个问题给出编号选项，用户回一个数字即可；
3. 选项中标注候选子流程的**产出物与耗时差异**，帮用户做决定；
4. 用户拒绝回答或说“你看着办” → 选最轻量的候选（单 Agent 优先于多 Agent）并声明选择理由。

**澄清话术模板**：

> 为了选对流程，确认两件事：
> 1️⃣ 你想要的是？① 快速结论（单Agent，几分钟） ② 深度团队研究（多Agent并行，更详尽但更久）
> 2️⃣ 你目前？① 未持有，考虑买入 ② 已持有，考虑去留
> 直接回复数字即可（如“1① 2②”）。

## 研究生命周期（标准链路）

子流程之间存在天然的上下游关系，报告路径遵循 `CLAUDE.md` 约定以保证数据链路连通：

```
筛选：industry-funnel / quality-screen / bottleneck-hunter
  → 深研：investment-research 或 investment-team（+ management-deep-dive 纵深）
    → 买前核对：investment-checklist
      → 买入后：thesis-tracker（建论文）
        → 持有期：earnings-review（每季）/ news-pulse（异动时）/ thesis-drift（定期）
          → 退出决策：exit-review（红线触发或想卖时）
组合层：portfolio-review（每季度，跨标的）
```

路由时若发现用户处于链路中段（如已持仓但无论文），主动提示补齐上游环节。

## 错误处理与用户反馈

遇到问题时不静默失败、不臆测填坑，按下表给用户明确的状态与出路：

| 场景 | 处理方式 | 向用户反馈的内容 |
|------|---------|----------------|
| 意图不明 | 用「智能澄清机制」，最多 2 问 | 编号选项，一次问完 |
| 参数缺失（公司名/时间窗/持仓清单） | 先补齐再启动，不带缺口执行 | “需要你补充：XXX，才能保证结果质量” |
| 超出范围（如短线择时、加密货币预测、期权策略） | 如实说明不做，给最接近的替代 | “本技能不做短线预测（价值投资定位）；最接近的是 XXX，要执行吗？” |
| 子流程中途失败（子 Agent 超时/异常） | 用已完成部分出降级版结论，标注缺失维度 | “N-1 个维度已完成，XX 维度失败，报告已降级输出并标注” |
| 工具调用失败（网络/参数） | 按 `skills/financial-data/references/verification-playbook.md` 降级：修正重试 → WebSearch 双源人工比对 | 报告中标注“未经工具验算”，不静默跳过验证 |
| 数据不足（冷门股/新上市） | 不用推测填满，按各子流程的 C 级信息稀缺模式执行 | “该公司公开信息稀缺，已切换第一性原理模式，结论置信度见报告声明” |
| 抽检反复打回 | 同一数据点连续 2 次打回且确认为口径差异 → 标注口径后人工放行 | 说明哪个数据点、两源数值与口径原因 |
| 报告写入失败（目录不存在/无权限） | 先创建目录重试；仍失败则输出到对话中 | 告知未能落盘及原因，给出建议路径 |

## 执行环境自检与兼容性（路由后、启动前执行）

不同 agent 环境的工具能力不同，启动多 Agent 类子流程前先自检，避免中途卡死：

| 检查项 | 具备时 | 缺失时的降级策略 |
|--------|--------|----------------|
| Team 协作工具（TeamCreate/SendMessage 等） | 按子流程原文执行 | 降级为并行子代理（同一消息并行启动，直接返回报告） |
| 并行子代理（Task/Agent 工具） | 并行执行 | 单 Agent 顺序执行各角色，报告标注“非真正并行” |
| WebSearch/WebFetch | 正常取数 | 告知用户无法联网取数，仅基于用户提供的资料分析并标注局限 |
| Bash + Python3 | 调用 tools/ 验算 | 降级为双源人工比对，报告标注“未经工具验算” |

降级发生时**在启动前告知用户**（一句话：“当前环境无 XX，将用 XX 方式执行，结果质量不受影响/会慢一些”），各子流程内的「环境降级约定」与本节一致。

## 启动与进度反馈约定

1. **启动时**：明确告知“将执行 `skills/<名称>/SKILL.md`”+ 产出物路径 + 预计耗时，然后打开该文件严格按其流程运行。
2. **执行中**（多 Agent/长流程）：每完成一个阶段向用户更新一次进度（已完成角色 + 该阶段 3-5 条核心发现），不要长时间无输出。
3. **完成时**：先给一段不超过 5 行的核心结论摘要（结论 + 关键数据 + 下一步建议），再给报告文件路径；不要只丢一个文件路径了事。
4. **下一步引导**：按「研究生命周期」提示自然的后续动作（如研究完→“要不要过一遍买入前 Checklist？”），但不强制。
5. **扩展**：新增能力只需在 `skills/` 下新建 `<名称>/SKILL.md`，并在上方路由表补充一行即可。
